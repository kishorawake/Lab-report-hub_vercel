# Lab Report Hub

AI-powered lab report analyzer — upload a blood/lab report PDF or image and get instant analysis, health scores, abnormal findings, and actionable recommendations.

## Stack

- **Framework:** TanStack Start (SSR) + TanStack Router (file-based routing)
- **UI:** React 19 + Tailwind CSS v4 + shadcn/ui (Radix primitives)
- **Animations:** Framer Motion + GSAP
- **Charts:** Recharts
- **PDF parsing:** pdfjs-dist (client-side, no server upload)
- **OCR:** Tesseract.js (client-side, no server upload)
- **TTS:** Google Translate TTS proxy (via `/api/tts` edge function)
- **Build:** Vite 7

## Getting Started

```bash
npm install
npm run dev        # → http://localhost:3000
npm run build      # production build → dist/client/ + dist/server/
npm run preview    # preview production build locally
```

## Deployment (Vercel)

Push to GitHub and connect to Vercel. The included `vercel.json` configures everything:

- **Build command:** `npm run build`
- **Output directory:** `dist/client` (static client bundle)
- **TTS proxy:** Served via `api/tts.ts` (Vercel Edge Function)

## Architecture Notes

### PDF / OCR Parsing
Lab report parsing is done **entirely client-side**:
- PDF text extraction via `pdfjs-dist`
- Image OCR via `tesseract.js`
- No files are uploaded to any server — privacy by design

### Lab Analysis Engine (`src/services/labAnalyzer.ts`)
The analysis engine uses **regex-based local parsing** against a reference table of normal ranges (`src/services/normalRanges.ts`). It does **not** call any AI API. When structured values can't be extracted, it falls back to demo data.

> **To add real AI analysis:** Wire `analyzeLabReport()` to a server function that calls the Anthropic/Claude or OpenAI API with the extracted text and asks for structured JSON output (test name, value, unit, interpretation, advice). Keep the API key server-side only.

### TTS Route
The `/api/tts` endpoint proxies Google Translate TTS to bypass browser CORS restrictions. It exists as both:
1. `src/routes/api.tts.ts` — TanStack Start server route (used in SSR/full-stack mode)
2. `api/tts.ts` — Vercel Edge Function (used in Vercel static deployment mode)

### De-identification (`src/services/deidentify.ts`)
Patient names, dates, and identifiers are stripped from extracted text before any processing.

├── components/          # UI components

├── routes/              # File-based routes (TanStack Router)
│   ├── __root.tsx       # Root layout + <head> metadata

│   └── api.tts.ts       # TTS proxy server route

│   ├── labAnalyzer.ts   # Lab report analysis pipeline
